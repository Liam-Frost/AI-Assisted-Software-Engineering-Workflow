# Acceptance Tests

## Purpose

This document defines **how we decide whether the system is “done.”**

It translates requirements into verifiable acceptance criteria.
If a requirement is not testable here, it is not complete.

---

## 1. Acceptance Principles

- Acceptance criteria must be objective
- “Looks correct” is not acceptable
- Manual and automated checks are both allowed
- Each criterion should map to one or more requirements

# 验收测试（ACCEPTANCE_TESTS）

## 文档目的

本文档定义**如何判断系统是否“完成”。**

它将需求转化为可验证的验收标准。
如果某个需求无法在这里被验证，则该需求不完整。

---

## 1. 验收原则

- 验收标准必须客观
- “看起来对了”不是验收条件
- 允许人工和自动化检查
- 每条验收标准应对应一条或多条需求
